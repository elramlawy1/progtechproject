ğŸ” FOR DEFENSE - KEY EXPLANATIONS:
---------------------------------

Q: "Explain your OO design"
A: The project uses layered architecture:
   - Model layer (Board, Game, Move, CellState)
   - AI layer (strategy pattern for extensibility)
   - Database layer (singleton pattern for resource management)
   - UI layer (CommandLineInterface)
   Each class has single responsibility, methods are well-encapsulated.

Q: "How does win detection work?"
A: After each move, Board.checkWinner() examines the position in 4 directions
   (horizontal, vertical, two diagonals). For each direction, it counts
   consecutive pieces. If 5 or more consecutive pieces found, player wins.
   The algorithm is in checkDirection() method - very straightforward!

Q: "Why did you choose these design patterns?"
A: 
   - Strategy: Makes AI extensible (can easily add SmartAI, DefensiveAI)
   - Singleton: DatabaseManager needs exactly one instance to manage connections
   - Facade: Game class hides board complexity from UI, easier to use
